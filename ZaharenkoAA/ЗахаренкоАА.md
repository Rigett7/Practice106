Описание проекта: Консольное приложение для управления списком задач

Описание созданного workflow:

Триггеры
push в ветку main – при каждом отправлении изменений в главную ветку.
pull_request в ветку main – при создании или обновлении запроса на слияние (PR) в главную ветку.

Шаги (steps)
Checkout code
Использует действие actions/checkout@v4 для клонирования репозитория на раннер.
Set up Python
Использует actions/setup-python@v5 для установки Python версии 3.9. Это гарантирует единую версию интерпретатора для всех запусков.
Install dependencies
Проверяет наличие файла requirements.txt и, если он есть, устанавливает зависимости через pip. В данном проекте файл пуст, так как используются только встроенные модули, но шаг оставлен для масштабируемости.
Run tests
Запускает все тесты, расположенные в папке tests/, с использованием встроенного модуля unittest. Команда:
python -m unittest discover -s tests -p "*.py".
Если хотя бы один тест падает, шаг завершается с ошибкой (ненулевой код возврата), и весь workflow получает статус «failure».
